# 图片框景

当国风输出包含照片、用户提供 AI 图、图片占位、截图、展品、菜品、地图或历史材料时，使用本文件。若任务涉及真实品牌、产品、人物、地点、活动、空间、榜单、工具、商业介绍、课程证据或内容科普，先读 `references/asset-workflow.md` 列素材 slot 和 provider，再用本文件处理图片框景、图注和版式。

## 通用规则

- 先做图片前置判断：没有真实照片、截图、展品、菜品、地图、产品图、人物图、空间图、活动图、服饰图或品牌案例图时，输出是否会失败。
- 图片前置现在服从素材前置：先在 `asset-manifest.md` 中列出对应 slot、类型、关键词、provider、状态和评分，并把最终决策写入 `guofeng-asset-spec.md`，再进入版式。
- **资产 > 风格。** 地点、展品、菜品、产品、UI、品牌案例和榜单对象的真实视觉证据优先于纹样、颜色和装饰构图；找不到 8/10 核心素材时使用诚实占位，不用泛图或 CSS 剪影凑数。
- **严禁使用 AI 图片生成工具（generate_image、DALL-E、Midjourney 等）替代真实照片。** 地点、景点、建筑、展品、菜品、产品、人物和空间必须使用真实照片或诚实占位，不要用 AI 生成的仿真照片。AI 生成的仿真照片会导致用户对真实场景产生错误期待。AI 生成图片仅允许用于：纯装饰纹理、抽象背景、或用户明确要求 AI 风格插画的场景。
- 涉及公共场所（景点、博物馆、寺庙、公园、历史街区、运河等）时，**默认主动联网搜索真实照片并下载到项目 `assets/images/`**。不需要等用户"明确允许"——公共场所的照片是内容核心；用户拒绝后再改用占位。
- 运行环境不支持联网、Python/Node 不可用或 agent 没有搜索/下载能力时，**在交付说明中提供图片获取指引**：列出每张图片的推荐搜索关键词、建议来源网站（如官方旅游网站、官网/政府/文旅/博物馆页面、国内图片搜索线索、Unsplash、Pexels、Wikimedia Commons）和建议尺寸比例，让用户自行获取并替换占位。
- 地点、展品、菜品、产品、服饰、空间、活动、人物、路线和品牌案例等内容型页面，默认真实图片是重要素材；拿不准时按“图片重要”处理。
- 真实图片决定成败但缺失时，用诚实图片位、占位或素材清单；不要编造图片来源，也不要设计在线图片生成服务流程。
- 缺图时不要静默交付纯占位版。交付说明必须列出中文素材清单，并说明哪些位置现在是“图片待补”。
- 启动联网搜索图片时，图片必须下载到输出项目的 `assets/images/`，HTML/CSS 使用相对路径引用。
- 不要热链远程图片 URL；不要把远程图片地址直接写进 `src` 或 `background-image`。
- 搜索下载的图片需要在 `assets/sources/asset-manifest.md`、`asset-candidates.json`、`guofeng-asset-spec.md` 和 `image-sources.md` 记录 slot、候选、来源、用途和状态。
- 版权不明图片只能标“来源待确认”或“仅占位参考”，不要写成已授权素材。
- 图片内容必须比装饰更重要。
- 用户需要查看对象、菜品、地点、路线或展品时，避免暗、糊、过度裁切的图片处理。
- 图注靠近图片，并保持可读。
- 不要把 dense 纹样放在细节照片后面。
- 源图较弱时，缩小图片并强化字体层级，不要硬拉满屏。

## 文化教育

| 内容 | 框景 | 图注需要 |
| --- | --- | --- |
| 博物馆展品 | 居中或图文并列，中性背景 | 名称、年代、材质、来源或语境 |
| 历史人物/事件 | 肖像或时间线裁切，安静边框 | 日期、身份、意义 |
| 诗词/文学 | 图片作为低对比氛围或小插图 | 题名、作者、朝代、解读 |
| 非遗/工艺 | 分步骤网格 | 工具、材料、流程、关键细节 |
| 课程截图 | 完整可读截图 | 说明学习者应看哪里 |

## 商业场景

| 内容 | 框景 | 实用信息 |
| --- | --- | --- |
| 入境游 | 真实地点照片，清楚 skyline/street/detail | 地点、路线、时间、票务、集合点 |
| 行程 | 地图式路线 + 地点卡片 | 天数、时间、交通、时长 |
| 地方餐饮 | 菜品照片，干净裁切 | 菜名、食材、价格、辣度/过敏提示 |
| 茶、酒、香氛、礼品 | 产品细节 + 材质纹理 | 产地、用法、购买或咨询行动 |
| 活动页 | 场地或活动图片 | 日期、时间、地址、报名行动 |

## 素材清单协议

当真实图片缺失时，在交付说明中列出：

- **必须补**：缺了会影响理解、信任或转化的图片，例如景点主视觉、展品正面、招牌菜、产品包装、活动场地、人物头像或品牌案例截图。
- **建议补**：能提升质感但不阻塞初稿的图片，例如环境细节、纹理、过程照、局部特写、氛围照。
- **当前占位**：页面里哪些 image slot 使用了占位，写明对应主题和建议比例，例如 `16:9 hero`、`4:3 展品图`、`1:1 菜品卡片`。
- **来源状态**：用户提供、可公开搜索、待授权、待拍摄或待确认。没有来源时不要写成已获得授权。

## 搜索下载协议（素材前置默认搜索）

涉及公共场所、地标、自然景观、历史建筑、可公开搜索的展品、品牌、产品、人物、空间、活动、榜单对象或工具时，**默认先按 `references/asset-workflow.md` 采集真实素材**。仅在以下情况不搜索：用户明确拒绝、运行环境不支持联网、或涉及非公开素材且用户要求不要联网。

搜索时：

- 优先按素材类型选择 source profile 和渠道矩阵：国内题材走 `domestic lane`，国际/开放版权题材走 `international lane`，不确定时走 `balanced`。Logo 走 SVGL / Simple Icons / Google favicon / 官方 brand 页；UI 走 App Store / Google Play / 官网截图 / 官方视频截帧；博物馆/历史内容可补 Met Museum Open Access 和 Art Institute of Chicago API；Unsplash/Pexels 只作为内容相关摄影线索。国内强平台只作为线索源，Wikimedia 只作为有限预算的国际开放源。
- 自动采集优先使用 Python 标准库脚本，不依赖 Node、npm 或 pip：
  ```bash
  python3 scripts/fetch-guofeng-assets.py --slot "Hero 西湖主视觉" --type place --query "西湖 杭州 全景" "West Lake Hangzhou panorama" --out site/assets/images --source-profile domestic --count 10 --pick 2
  ```
- `scripts/fetch-guofeng-images.py` 保留为 Wikimedia 快捷入口；完整生成默认使用 `scripts/fetch-guofeng-assets.py`。
- 下载到 `assets/images/`，文件名使用可读 slug，例如 `west-lake-hero.jpg`、`product-packaging-01.webp`。
- 在 `assets/sources/asset-manifest.md`、`asset-candidates.json`、`guofeng-asset-spec.md` 和 `image-sources.md` 记录：slot、候选、本地文件、原始来源 URL、页面用途、来源状态、检索日期。
- 不确定授权时，页面可作为占位初稿使用，但交付说明必须提醒"来源待确认"。

无法联网时（fallback）：

- 在 HTML 中使用诚实占位（带说明文字的灰色/纹样区块，标注建议比例如 `16:9 hero`、`4:3 景点图`）。
- 在交付说明中为每个占位图列出：**推荐搜索关键词**（中英文各一）、**建议来源网站**（如 `unsplash.com/s/photos/west-lake`、`commons.wikimedia.org`）、**建议尺寸比例**。
- 示例：`hero 占位 → 搜索"西湖全景 / West Lake panorama"，建议 Unsplash 或 Wikimedia，16:9，≥1920px 宽`。

## 取图四步与评分

内容图必需时，按以下四步执行后再进入版式设计：

1. **构造精确关键词**：地点用「对象 + 城市/地区」，展品用「名称 + 馆藏/朝代/材质」，菜品用「菜名 + 餐厅/地方」，品牌案例用「品牌名 + product/UI/logo」。
2. **下载候选素材**：优先用 `scripts/fetch-guofeng-assets.py` 按 provider 分级采集；脚本不可用或无结果时，改用浏览器/WebSearch 找官方来源或 B 级线索源。
3. **8/10 评分**：只使用评分达到 8 分的素材；低于 8 分宁可占位，不硬塞泛图。
4. **写来源记录**：所有使用或候选图片都记录到 `assets/sources/asset-manifest.md`、`asset-candidates.json`、`guofeng-asset-spec.md` 和 `image-sources.md`，最终交付说明标注 `已验证`、`待用户确认` 或 `占位待补`。

8/10 评分表：

| 维度 | 通过标准 | 不通过时 |
| --- | --- | --- |
| 地点/对象匹配 | 来源页标题、描述、分类或官方页面能确认目标地点/展品/菜品/产品 | 不使用 |
| 来源可信度 | 官方、Wikimedia Commons、明确免费图库或用户提供 | 标 `待用户确认` 或占位 |
| 分辨率 | hero 图建议 ≥1600px 宽；卡片图建议 ≥1000px 宽 | 缩小使用或占位 |
| 构图可用性 | 主体清楚、不过暗不糊、不被严重裁切，有留白可排文字 | 换图 |
| 版权清晰度 | 许可、作者或来源状态可记录 | 不写成已授权 |

交付说明必须列出每张内容图状态：

- `已验证`：来源能确认地点/对象匹配，授权状态已记录。
- `待用户确认`：视觉可用但来源未完全确认，仅作为初稿参考。
- `占位待补`：未找到 8/10 素材，页面使用诚实占位和素材清单。

## 图片搜索验证协议

搜索并下载图片后，必须执行验证。**未通过验证的图片不得使用，改用诚实占位。**

### 搜索关键词构造

- 必须使用「具体地名 + 城市/地区名」的精确组合，不要用泛关键词。
- 每个关键词同时构造中文和英文版本，优先用中文搜索中国地点。
- 示例：
  - ✅ `西湖 杭州` / `West Lake Hangzhou`
  - ✅ `灵隐寺 杭州` / `Lingyin Temple Hangzhou`
  - ✅ `龙井茶园 杭州` / `Longjing Tea Plantation Hangzhou`
  - ❌ `lake scenery` / `temple` / `tea field`（太泛，会匹配到其他地点）

### 来源页验证

下载前必须检查图片来源页面的以下信息，至少一项能确认地点匹配：

- 页面标题或图片标题包含目标地点名称
- 图片描述/alt 文字明确提及目标地点
- 来源为目标地点的官方旅游/文化机构网站
- Wikimedia 图片的分类或描述包含正确地理标签
- Unsplash/Pexels 图片标签包含正确地点名

**如果无法从来源页确认图片确实拍摄于目标地点，不要使用该图片。**

### 宁缺毋错

- 找不到确认匹配的真实照片时，使用诚实占位 + 素材清单，**不要硬塞一张看起来像但无法确认的图片**。
- 一张错误地点的照片比一个灰色占位框更有害——它会损害整个页面的信任感。
- 交付说明中对每张使用的图片标注验证状态：`已验证`（来源确认地点匹配）、`待用户确认`（视觉相似但来源未明确标注地点）或 `占位待补`（未找到 8/10 素材）。

## Pattern Pairing

- `tile.webp` 只作为图片周围低透明氛围。
- `strip-horizontal.webp` 用于 caption band 或 section break。
- `strip-vertical.webp` 用作图片组旁的侧边纹样。
- `cover-accent.webp` 用在网站 hero 或 deck cover，前提是文字有足够留白。

## 常见错误

- **使用 AI 图片生成工具（generate_image / DALL-E 等）生成地标、景点、建筑、菜品或展品的仿真照片来替代真实照片。**
- **搜索图片时使用泛关键词（如 lake、temple），导致下载的图片不是目标地点。**
- 还没判断真实视觉证据是否必要，就先选纹样和颜色。
- 缺少关键图片却只在 HTML 里放灰框，没有在交付说明里列素材清单。
- 搜索后仍然热链远程图片，没有下载到 `assets/images/`。
- 下载了图片但没有记录 `assets/sources/asset-manifest.md`、`asset-candidates.json`、`guofeng-asset-spec.md` 或 `image-sources.md`。
- 给装饰 banner、文章封面或无关卡片滥用 Unsplash/Pexels stock photo。
- 把小红书、点评、淘宝、抖音、公众号等强平台写成默认自动抓取，而不是线索源或浏览器辅助。
- 旅游页有诗意但缺路线、地点、时间或预约行动。
- 菜单页中装饰字体或纹样影响价格和菜品说明阅读。
- 博物馆页里展品比纹样还小。
- 课程页图片很美，但知识点不清楚。
