# 生成工作流

生成完整国风网页或 HTML PPT 前，先用本文件确定稳定方向。目标不是先挑纹样，而是先弄清楚“为谁、做什么、凭什么、让用户下一步做什么”。

## 全场景 Preflight：开始前的十个判断

> Preflight 是必须与用户交互的阶段，不是 agent 内部的自我判断。以下问题必须以中文向用户提出，等待回答后再开始生成。（平台提示：GPT 系列模型倾向跳过提问直接生成——请在回复开头用加粗文字标注「⏸ 生成前需要您确认以下问题」，强化停顿信号。）

1. **受众**：谁会看，中文用户、外国游客、课堂听众、商业客户、活动参与者还是内部汇报对象。
2. **输出规模**：单页落地页、内容型长页、多页面小站、HTML deck 页数、演讲型/课程型/发布会型 PPT。
3. **输出类型**：website page、multi-page static site、HTML PPT deck，或单个 page/section。
4. **用途**：文化教育、商业转化、品牌展示、活动传播。
5. **核心信息**：必须让用户记住、理解或行动的 1-3 个重点。
6. **书写系统**：`zh-Hans`、`zh-Hant`、English 或 bilingual。
7. **国风方向**：`ink-wash`、`porcelain-blue`、`tang-vermilion-gold`、`jade-green`、`dunhuang-earth`、`night-lacquer`、`hua-chao-rouge`。
8. **资料状态**：只有文字、已有图片、已有截图、已有 logo/产品图/UI、已有 AI 图、需要素材位、只能先用占位。
9. **素材前置**：先判断真实照片、logo、展品图、菜品图、路线图、产品图、人物图、空间图、活动图、服饰图、UI 截图或品牌案例图是否决定成败。
10. **事实风险与 Fallback mapping**：日期、价格、地址、路线、来源、展品信息、菜单、规格、产品 claim、历史细节必须核实或标缺失；未列明场景先映射到最接近的用途和信息任务，再选版式。

## 通用开工前问题模板

用户需求含糊时，**必须用中文一次性列出 5 组问题并等待回答**。问题要像设计顾问在开工前对齐方向：让用户批量回答，不要一问一答拖慢节奏；也不要机械照搬所有例子，只保留会改变版式、语气、语言、素材处理或交付规模的关键问题。

推荐开场：

```text
好的！这个题目适合做成国风视觉产出。开工前先快速对齐几个关键问题，你批量答就行：
```

首轮问题必须覆盖：

1. **目标受众 & 使用场景**
   - 给谁看？中文用户、外国游客、课堂听众、商业客户、活动参与者、内部汇报对象，还是某类垂直人群？
   - 用途是什么？品牌展示、商业转化、知识讲解、活动传播、城市导览、菜单展示、课程演示、展陈说明、产品介绍还是内部汇报？
2. **输出形式 & 规模**
   - 要单页长页、多页面小站、HTML PPT deck、单个页面区块，还是先做一个首页/封面初稿？
   - 如果是 PPT，确认页数和类型：演讲型、课程型、发布会型、路演型或展陈导览型。
3. **核心内容板块**
   - 用户必须看到什么？例如产品卖点、课程大纲、展品信息、路线、菜单、活动流程、品牌故事、人物介绍、案例证据、CTA。
   - 有没有必须保留或禁止出现的内容？
4. **素材 & 事实状态**
   - 有没有现成文案、图片、logo、产品图、UI 截图、菜单、展品资料、地址、价格、时间、路线、联系方式或来源？
   - 缺失内容是联网查证、用 Node 素材脚本或浏览器辅助采集到本地、标 `待确认`，还是先用诚实占位和素材清单？
5. **参考风格 & 默认授权**
   - 有没有参考 URL、截图、品牌案例、logo、品牌色、字体规范，或一句“就要那种感觉”？
   - 如果都没有，用户可以说“你看着办”。只有用户明确授权时，才可以按默认判断推进，并在交付说明中写清 `假设`、`占位内容`、`待确认事实` 和 `下一步资料`。

场景化替换规则：

- 文旅、餐饮、博物馆、寺庙、园林、街区、民宿和活动页：默认说明会主动采集公开真实照片并下载到本地；用户拒绝、环境不可用或无法验证来源时列素材清单，不用 AI 仿真照片替代。
- 品牌、产品、餐饮、茶酒香器、伴手礼、工具榜单和商业展示页：默认询问 logo、产品图、UI 截图、品牌色、字体规范、参考页面、核心卖点、价格/规格/购买或咨询方式。
- 课程、讲座、知识科普和文化教育页：默认询问听众基础、课程目标、章节结构、讲义/资料来源、是否需要中英双语或注释。
- HTML PPT：默认询问页数、演讲场景、讲述节奏、封面/章节/结尾要求和是否需要投屏演示。
- 多页面小站：默认询问页面清单、导航层级、首页优先级和每页必须完成的行动。

**仅当用户明确说"不用问了直接做"、"你来决定"、"给个初稿先"或类似指令时**，才允许跳过提问，使用下方默认值并在交付说明中显眼标注所有假设。用户没有回应 ≠ 用户说了"不用问"。

## 静态站点独立交付

完整输出默认是可搬走的静态站点项目，不是只生成一个孤立 HTML 文件。

单页或内容型长页默认结构：

```text
site-root/
  index.html
  assets/
    patterns/
    images/
    css/
    sources/
```

多页面小站默认结构：

```text
site-root/
  index.html
  pages/
  assets/
    patterns/
    images/
    css/
    sources/
```

HTML PPT 默认结构：

```text
site-root/
  deck.html
  assets/
    patterns/
    images/
    runtime/
    sources/
```

规则：

- 用到的纹样必须复制到输出项目的 `assets/patterns/`。
- 用户提供、脚本采集或浏览器辅助保存的图片必须放到 `assets/images/`，来源记录写入 `assets/sources/asset-manifest.md`、`asset-candidates.json`、`asset-leads.md` 或 `image-sources.md`。
- HTML PPT runtime 必须放到输出项目的 `assets/runtime/` 或使用项目内相对路径。
- HTML/CSS 只引用相对路径，例如 `assets/patterns/...`、`assets/images/...`、`assets/runtime/...`。
- 不要引用 `<skill-root>`、用户主目录、系统绝对路径或 agent 缓存目录。

## 输出规模判断

- 用户说“网站”时，不默认等于单页。先区分：单页落地页、内容型长页、多页面小站。
- 用户说“PPT”时，不默认等于 3 页模板。先区分：HTML deck 页数、演讲型、课程型、发布会型。
- 用户说“页面”时，默认单页，但仍需判断是导览页、产品页、菜单页、展品页、活动页还是品牌首页。
- 仅当用户明确跳过提问时：网站默认生成"单页长页初稿"，PPT 默认生成"5-7 页 HTML deck 初稿"，页面默认生成"单页初稿"。交付说明必须显眼标注这是初稿规模，不是多页面网站或最终完整 deck。

## 三项必问（STOP-GATE）

完整输出前**必须向用户提出以下三项问题**，并等待回答。这是一个 STOP-GATE：**在用户回答之前，不要开始生成 HTML。**

这三项是最低门槛，不替代「通用开工前问题模板」。宽泛需求首轮仍应一次性覆盖五组问题；如果用户材料已经很完整，也必须确认下列三项已经明确。

- **输出规模**：单页长页、多页面小站、HTML PPT deck，或单个 page/section。
- **国风主题**：从 2-3 个适合场景的 theme/direction 中推荐一个，并说明理由。
- **素材策略**：用户提供素材、联网采集真实素材并下载到本地（公共场所、具名品牌/产品/工具默认选择此项，用户拒绝才占位）、B 级平台只做浏览器辅助线索，或先用诚实占位和素材清单。

不要静默默认任何选项。**用户未回答时不要生成完整 HTML。**

> 唯一例外：用户明确指示"不用问了"、"按你的想法做"、"给个初稿先"时，可以使用默认值，但交付说明必须显眼标注所有假设。

## 方向顾问：撞 / 借 / 请

当用户只说“做一个国风页面”“做一个中式 PPT”或只给一份宽泛材料时，先帮用户选方向。

- **撞 / contrast**：题材容易变安全国风时，用有意义的反差。例：现代科技配 `porcelain-blue` 的冷静秩序，高端香氛配 `night-lacquer`，文旅线路配 `dunhuang-earth` 加干净编辑结构。
- **借 / borrow**：陌生场景借用相邻真实场景。例：寺庙导览借博物馆展签和入境游结构，新中式香氛借礼品品牌页，文化路线借 itinerary。
- **请 / role lens**：先请一个角色进入判断。例：策展人、旅行编辑、品牌设计师、课程制作人、餐厅策划、活动制作人。

避免“安全国风”：米色纸、红印章、随机边框和空泛诗句。国风方向必须影响信息层级、字体、节奏和图片处理，而不只是装饰。

用户要求“给几个方向”“推荐风格”“你来定方向”时，至少给 3 个中文方向草案；不必每次生成 3 个完整 HTML，但必须让用户看懂每个方向的受众、信息结构、视觉气质和素材需求。

## 事实先于假设

写最终文案前，先分清事实和氛围：

- 会影响用户行动的事实必须核实或向用户确认：日期、时间、地址、价格、路线、票务、联系方式、预约方式、来源、展品 metadata、菜单项、产品 claim、历史陈述。
- 来源缺失时，写 `待确认` / `to confirm`、留空，或改成不承诺事实的描述。
- 不要用漂亮文案制造权威感，不要编造排名、引用、物流、路线或历史背景。

## 字体依赖

- Google Fonts 中存在的字体（例如 Noto 系列）可以通过 Google Fonts CSS API 加载。
- Google Fonts API 必须使用 `https://fonts.googleapis.com/css2...`。
- 不要手写 `fonts.gstatic.com/s/...` 字体文件 URL。
- 装饰字体 `ZhaohuaMinA`、`KingHwaOldSong-MN` 通过 `https://cdn.itbxm.com/fonts/.../result.css` 加载。
- 不要打包 TTF、手写 `@font-face` 字体文件 URL，或引用旧的 `fonts.itbxm.com` 字体地址。

## 默认值

- 输出规模：用户明确跳过提问时，网站默认"单页长页初稿"，PPT 默认"5-7 页 HTML deck 初稿"，页面默认"单页初稿"。
- 输出类型：单页或长页网站用 `web-page.html`，演示用 `ppt-deck.html`；多页面小站需先规划页面清单，再为每页复用模板和组件。
- 项目结构：完整输出默认包含 HTML 与 `assets/`；用到的纹样、图片、素材来源记录和 runtime 都复制到输出项目内。
- 用途：文化主题默认文化教育；产品、地点、服务、机构默认品牌展示。
- 书写系统：从用户材料推断；没有材料时默认 `zh-Hans`。
- 主题：文化教育默认 `ink-wash`；精致文化/科技默认 `porcelain-blue`；节庆/仪式默认 `tang-vermilion-gold`；茶、自然、养生默认 `jade-green`；历史、遗产、旅行默认 `dunhuang-earth`；高端品牌、封面、戏剧化章节默认 `night-lacquer`；花朝、香氛、婚宴、春日市集默认 `hua-chao-rouge`。
- 纹样密度：阅读区用 sparse 或 medium；dense 只用于 hero、deck cover、章节页或低透明背景。
- 未列明文化内容默认文化教育。
- 未列明地点、服务、产品或组织默认品牌展示。
- 未列明活动默认活动传播。
- 未列明销售、预约、点单或获客页面默认商业转化。

## Fallback Mapping

场景或材料类型不熟悉时，不要等完美匹配。

- 先提取受众、交付物、核心信息、资料状态、必须保留内容和禁止内容。
- 用户只给主题或文档时，只问会影响版式、语气、语言或资产处理的问题。
- 用户已回答 Preflight 或明确跳过提问后，从 `web-page.html` 或 `ppt-deck.html` 开始，再用已注册 layout 和 component 改造。
- 缺素材时，用诚实图片位、占位或素材清单；不要编造图片、logo、UI 截图或来源。
- 缺数据时，不编造价格、日期、地址、路线、引用、排名、claim 或历史事实；标 `待确认` 或删除事实承诺。
- 用户要求后台、上传、支付、实时地图、预约、登录或图片生成时，保持静态输出，提供 CTA 或占位文案替代。

## 交付说明

完整生成后必须附简短中文说明，包含：

- **输出规模**：单页、内容型长页、多页面小站规划、HTML deck 页数，或单个 page/section。
- **国风方向**：采用 `撞`、`借` 或 `请` 中哪一路，以及选用的 theme。
- **素材状态**：使用了哪些真实素材，哪些是占位，哪些图片/logo/UI/产品图待补。
- **待确认事实**：缺少来源的时间、地点、价格、路线、菜单、规格、引用、历史细节或产品 claim。
- **下一步资料**：用户补充什么最能提升效果。

## 用途规则

### 文化教育

用于课程、博物馆、历史、诗词、文学、非遗/工艺、传统节日、城市文化、学术讲座和公共文化传播。

- 知识层级优先：标题 -> 核心概念 -> 证据/例子 -> 注释/来源 -> takeaway。
- 段落必须可读；纹样用于框定内容，不与内容抢注意力。
- 优先使用 `L03 Poem Quote`、`L07 Timeline`、`L11 Course Explainer`、`L12 Museum Label`。
- CTA 可选，只用于查看档案、下载讲义、继续阅读或报名讲座。

### 商业应用

用于入境游、地方餐饮、文旅推广、本地品牌、茶酒香器、民宿、伴手礼和活动报名。

- 转化信息优先：地点、时间、价格、路线、菜单、预约、导航、联系方式。
- 国风氛围用于提升信任和识别度，不要遮挡实用信息。
- 优先使用 `L05 Artifact Showcase`、`L06 Card Grid`、`L13 Travel Itinerary`、`L14 Local Food Menu`。
- CTA 必须清楚：预约、导航、看菜单、咨询、购买或报名。
